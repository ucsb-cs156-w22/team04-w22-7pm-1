Navbar related Acceptance Criteria
 There is a menu item PersonalSchedule (this is done on the Navbar)
 The menu item on the navbar only shows up when you are logged in.
 It has two items for Admins, List and Create
 It has one item for regular Users, List
 List routes to /personalschedule/list
 Create routes to /personalschedule/create
Page related acceptance criteria
 There are pages components under /frontend/src/main/pages/PersonalSchedule/ similar to those under /frontend/src/main/pages/Todos/ for the create and index pages.
 There are entries for these pages in the storybook.
 The List placeholder page is available at /personalschedule/list (this is done in App.js)
 The Create placeholder page is available at /personalschedule/create (this is done in App.js)
